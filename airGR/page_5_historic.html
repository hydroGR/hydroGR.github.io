Customization

It is possible to complement the airGR package with a new model. To do that, the main steps are the following:

  1. Write a model in Fortran or modify an existing Fortran code that is included in the airGR package
  2. Create specific R functions to the model (RunModel_MyNewModel(), TransfParam_MyNewModel())
  3. Two possibilities exist to include the new model: a. Create a dynamically loadable library (DLL; also known as shared object) ensuring compatibility with R and load it with the dyn.load() function b. Create a new package dependent on airGR which contains only your R model and functions of step 2
  4. If the new model requires model inputs that are not already present in airGR, you have to adapt the CreateInputsModel() and CreateRunOptions() functions (note: it is better to duplicate these functions and to modify them, to avoid any conflict when using original airGR models)
